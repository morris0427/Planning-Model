QUICK START:
============

1. Copy your data generation:
   cp your_blocks_generation.py data/blocks_world.py
   # Then wrap in BlocksWorldDataset class (see INTEGRATION_GUIDE.md)

2. Copy your model:
   cp your_model.py models/transformer.py
   # Then add weight_sharing parameter

3. Copy your training:
   cp your_train.py trainer.py
   # Then wrap in train(config) function

4. Test:
   python run_experiments.py --experiment blocks_world_standard

5. Run ablations:
   python run_experiments.py --sweep model_size --domain blocks_world


EXPERIMENT COMMANDS:
====================

# Single experiment
python run_experiments.py --experiment blocks_world_standard

# Model size sweep (6 experiments)
python run_experiments.py --sweep model_size --domain blocks_world

# Weight sharing ablation (4 experiments)
python run_experiments.py --sweep weight_sharing --domain blocks_world

# Full ablation (12 experiments)
python run_experiments.py --sweep full --domain blocks_world

# Productivity split
python run_experiments.py --sweep model_size --domain blocks_world --split productivity

# Analyze results
python analyze_results.py --sweep-file results/sweep_summary.json


WHAT EACH FILE DOES:
====================

config.py:
  - Defines model architectures (tiny/small/medium/large)
  - Defines data splits (in-distribution, productivity, etc.)
  - Provides preset configurations
  - Generates experiment sweeps automatically

run_experiments.py:
  - Runs experiments based on configs
  - Saves results to JSON
  - Tracks experiment metadata
  - CLI interface for easy execution

analyze_results.py:
  - Loads sweep results
  - Generates comparison tables
  - Identifies best configurations
  - Prints formatted output

data/base.py:
  - Abstract interface for datasets
  - Ensures consistency across domains
  - Factory pattern for easy creation

data/blocks_world.py (YOU CREATE):
  - Implements generate_problem() using SAW
  - Implements encode_sequence()
  - Implements decode_sequence()
  - Registers with factory

models/transformer.py (YOU CREATE):
  - Your transformer architecture
  - Adds weight_sharing support
  - Factory function for model creation

trainer.py (YOU CREATE):
  - Your training loop
  - Takes ExperimentConfig as input
  - Returns results dictionary
  - Saves checkpoints

BENEFITS:
=========

Before framework:
  - 12 separate scripts for full ablation
  - Manual result tracking in spreadsheets
  - Risk of inconsistent settings
  - Hard to add new experiments

After framework:
  - Single command: python run_experiments.py --sweep full --domain X
  - Automatic result tracking
  - Guaranteed consistent settings
  - Easy to extend
