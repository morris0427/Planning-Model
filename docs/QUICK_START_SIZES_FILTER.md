# QUICK START: Running Large Models Only

## ✅ **NEW: Single Script Solution**

**You asked for:** A way to run ONLY large models without a full sweep, using ONE consistent script.

**You got it!** ✓

---

## 🚀 **Simple Usage**

### **Run ONLY large models (8-Puzzle):**

```bash
cd experiments

python3 run_experiments.py \
  --sweep model_size \
  --domain eight_puzzle \
  --sizes large
```

**This will run:**
- `eight_puzzle_large_base` 
- `eight_puzzle_large_wm`

**Time:** ~4-6 hours (just 2 experiments)

---

### **Run ONLY large models (Blocks World):**

```bash
python3 run_experiments.py \
  --sweep model_size \
  --domain blocks_world \
  --sizes large
```

**This will run:**
- `blocks_world_large_base`
- `blocks_world_large_wm`

**Time:** ~4-6 hours (just 2 experiments)

---

### **Run ONLY large models (both domains):**

```bash
# 8-Puzzle
python3 run_experiments.py --sweep model_size --domain eight_puzzle --sizes large

# Blocks World
python3 run_experiments.py --sweep model_size --domain blocks_world --sizes large
```

**Time:** ~8-12 hours total (4 experiments)

---

## ✅ **Why This is Better**

**Old approach (run_large_models.py):**
- ✗ Separate script with different code path
- ✗ Risk of inconsistencies
- ✗ Confusing to maintain two scripts

**New approach (run_experiments.py --sizes):**
- ✅ Same script for all experiments
- ✅ Same code path = consistency guaranteed
- ✅ Explicit about what's running
- ✅ Flexible (can filter to any size combination)
